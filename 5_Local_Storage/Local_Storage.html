API

Application Programming Interface

It is an intermediary application that links a main service to an user.
The user can be another service, website, machine.

Example: Google Maps is the main service of maps and localization from Google. The user can be a website of a company located anywhere in the World. The API that access information from Google Maps can be installed in this company site to show its real location. The company website does not have any map, but it can show the map because the API links it to the main Map Service.

Cookies Storage

The ancient browsers and versions of browsers used only cookies tostore data.

One cookie can store up to 4 Kb of data.

Each page had a limit of 30 to 50 cookies.

Web Storage

Session Storage

Stores data until the session is closed. When you close the tab or the browser the information is erased.

The Session Storage can store up to 5 Mb.

Local Storage

Maintains the information even after closing the browser, there is no expiration date. To erase the information you need to clean the browser cache.

The Local Storage has no limit. It depends on your computer.

Security

We must not store sensitive information:

  • Username
  • Password
  • Credit card number

Arrays

Indexed Arrays

The values in an Indexed arrays are indexed using integers, starting in 0 (zero). It is like a one-column table.

Array = ["first", "second", "third"]

Associative Arrays

The values in an Associative arrays are indexed using keys, starting in 0 (zero). It is like a two-column table; the key is in the first column and the value is in the second.

Array = {"month":"January","day":"23","weekday":"Thursday","year":"2020"}

Only strings can be stored in Local and Session Storages.

Arrays must be converted to strings in order to be stored. The command to convert an array to string is JSON.stringify().

To retrieve an string and convert it back to an array, we must use the command JSON.parse().
